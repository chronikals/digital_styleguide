Adaptive layouts

Responsive design

Responsive design adapts user interfaces to multiple screen sizes. Content is scaled based on user needs, for example in applications that are rendered on different devices, or device-specific versions of the same application.

The intention is to make sure your layout is both situationally relevant and functionally appropriate for the device and task at hand. In order to make your interface responsive and decide how it will scale, these core issues must be considered:

1. Understand the content

What is the intention of the product and what is the main content? How do the content sections relate to each other and how well does each section scale? Not all content types scale equally well. Understanding the content and how it scales is the starting point for deciding how to adapt your design.

2. Know your formats

Which hardware formats will your product be used on? Is it just one or will there be multiple? Is there a typical set of formats/dimensions? Some software products are sold with specific hardware panels, while other products will be displayed on whatever the customers use.

3. Decide on breakpoints

Once you know your content and your formats you can decide which breakpoints your interface needs to scale well on all formats. At the breakpoint a different grid is applied, while in-between breakpoints the grid itself scales.

4. Adapt your content

You know your content, the formats, and the breakpoints; now you need to decide how your product will adapt to these constraints. Decide on which grid to use for which breakpoints, and how to lay out the content in each grid.

Decide on breakpoints

Grids establish an organizational layout system for content. A breakpoint is a specific resolution when a specific grid layout should be applied, while in-between breakpoints the grid itself scales. Breakpoints determine a flexible structure for layouts to adapt to screen sizes without designing one layout for each screen resolution.
