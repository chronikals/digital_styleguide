Responsive tables

When it comes to making tables responsive and adapting them to different hardware and screen resolutions, there is no one size fits all solution. Tables are good for housing a lot of data, but on small screens data comparison can be difficult because limited information can be displayed on the screen at once.

4 ways of adapting tables

This chapter describes 4 possible solutions for adapting tables to smaller screens:

  1. Hide the least important columns
  2. Freeze the first column or row
  3. Restructure the table
  4. Keep the full table - zoom in/out

These solutions may also be combined. Example: When scaling a window, the first adaptation can be Hide the least important columns, and when the window becomes too small, the next adaptation could be Restructure the table.

Which solution to choose?

The best solution depends on content, user goals and context of use. Answering these questions may help you decide:

  • What information is essential to understand the table content?
  • What information does the user care about the most?
  • What is necessary to be able to differentiate one piece of information from another?
  • Is it important to compare rows and columns?
  • Does everything need to be presented all at once or can some information be available only on request?

Checklist

To ensure good usability, make sure your table satisfies the following criteria when choosing a solution:

  • Is the essential information accessible regardless of device?
  • Will people understand the meaning of the table in all contexts?
  • Can people get to all of the information somehow?

1) Hide the least important columns

Prioritize which columns according to importance for the default case.

When a browser window size is reduced, or the interface is displayed on a smaller screen, the column width is gradually decreased to a defined minimum. When there’s not enough space to view all columns, the least important columns are hidden in prioritized order until only the essential ones remain.

In this example, we have considered the columns for parameter name, value and change essential. The 3 other columns will only be visible if the screen or window size is large enough.

2) Restructure the table

Flip table axis

When the user needs to compare entire rows, or view all data related to a single item at once, the table axis can be flipped on smaller screens. In this example, the header row becomes a header column and the rows are now columns that are horizontally scrollable.

Rearrange table rows

Another way to restructure the table, is to change the formatting so that all data related to a single row is rearranged into compact groups instead of spread across the row. This creates a taller table, requiring more vertical scrolling, but without the need for horizontal scrolling. It’s easier to browse the data without losing context. The downside is that it might be more difficult to compare data across rows.

3) Freeze the header column or row

The table header provides the key reference for the content of the table. Having it available at all times, helps the user maintain the overview - even when only parts of the table is displayed on smaller screens.

In this example, the left column header is fixed as the user scrolls the table horizontally.

4) Keep the full table - zoom in/out

In some cases, the best solution is to display the full table without adapting it to a smaller screen size. The user may be accustomed to viewing and interacting with the table on a larger screen, and prefer a recognizable table that gives him the same overview.

When keeping the full table unchanged, the user can zoom in and out to view relevant information.
