Menus

ABB bar menus

Accordions

Accordion menus makes the top-level navigation options continuously available, while highlighting the current area its sub-navigation options. As there is only one sub-level open at any time, this pattern is most appropriate when there is not excessive navigation between sub-sections.

Icons can be used on each item, but be aware that it might also lead to visual clutter. This is best used when there is a limited set of page types, with clearly distinguishable icons.

Tree views

Tree views can provide a lot of functionality, but can often lead to complex interaction and users getting lost. When using tree view navigation, strive to reduce number of levels and ensure the most used views are located in the highest levels. If your tree view menus become large and complex, consider splitting content into separate sections or use other interaction models for some content.

There are many graphical and interaction design choices to be made when using tree views, and there is no 'one size fits all'-solution that fits all cases. This chapter shows some alternative tree view menus designs within the ABB UI styling. Tree views require particular attention in each use case, and be sure to test the solution before it is too late to make changes.
