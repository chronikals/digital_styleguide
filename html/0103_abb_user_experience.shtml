ABB's user experience

What is user experience?

User experience (UX) can be defined as the overall experience you have when using or interacting with something. That “something” could literally be almost anything, but in this context it refers to the experience the user has in contact with ABB software. This includes issues such as:

  • Is the product easy to use?
  • Is it efficient in helping me in solving my task?
  • Does it behave as expected?
  • Is the look and behavior consistent throughout the application?
  • Does the user interface look professional and trustworthy?
  • Is it delightful to interact with?

Three reasons to focus on user experience

  1. Improve efficiency and reliability in use
  2. Avoid user errors
  3. Desirable products increase sales

A style guide vs. great user experience

The illustration is based on Stefan Klocek’s “Hierarchy of effort to fix a broken user experience”

This style guide aims to lay the foundation for visual simplification and consistency across ABB’s software portfolio. It also includes UI patterns and examples that can be reused. However, using a style guide does not guarantee great user experiences, or even consistent user experiences.

This style guide will help ABB advance up the two first levels in the UX maturity hierarchy, but to reach higher levels we as a company need a wider collaborative effort. In order to consistently and over time deliver great software that satisfies our customers, there must be a commitment to user-centered product design and UX research in all phases of product development. This includes thorough user insight into what really makes a difference for the safety or productivity of our end users, iterative product development and validating assumptions through end-user testing.

ABB software interfaces

ABB software is used on a wide variety of hardware formats and within different environments (control room, field, office, home). In addition, a variety of input devices, such as keyboard, mouse, touch screens, trackpads and embedded buttons are used.

The user interface must be adapted to format and context. This style guide intends to support product teams in delivering consisten and recognizable user experiences across all ABB software interfaces.
