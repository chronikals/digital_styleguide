Creating visualizations

There are two important rules of thumb when designing data visualizations:

1: Make sure the content is meaningful to the user

  • Use relevent data for the current situation and user task(s)
  • Select best possible range/scale
  • Provide enough contextual information

2: Draw attention to whatâ€™s important

  • Create a visual hierarchy of information and highlight the most important information
  • Avoid unneccessary effects and details

Visual hierarchy where essential information is given the highest priority:

Backgrounds (elements that as discretely as possible frame and group main content)

Support/secondary information (such as limits, setpoints, range. labels)

Primary information

Alerts and highlights

How to draw attention to the data

The basic properties of data visualization are called pre-attentive attributes. These are aspects that we detect very rapidly and accurately, such as differences in color, intensity, size, orientation and placement. Used consciously, these attributes help highlight and draw attention to specific data. Several attributes can be used together, but avoid combining too many at once (see illustration to the right).
