Table feature examples

The following section contains a selection of patterns for commonly used table features. For each individual case you must decide whether to use these features directly, modify them, or if other interaction patterns are better suited. If using different patterns, make sure to always follow the overall look and feel of the ABB UI style guide.

Alternating row colors

Used when the user needs to visually separate similar looking rows in a list, in order to match or compare data. Alternating row colors (or “zebra stripes”) can be especially useful for improving legibility in long and wide lists containing similar looking values or information. Be careful when using in tables with much vertical scrolling or when combining this with inline controls, sub-headers or other background color effects.

Sub-headers

Sub-headers with different background color can be used to label or separate groups of data in lists and tables.

Use of background color

Columns or cells with different background color can be used to emphasize or differentiate parts of a table, to enhance legibility.

Sorting

The blue line below a column header indicates that the table is sorted by this column. The arrow indicates whether the column is sorted in ascending or descending order. Clicking this column header, will reverse the list (and arrow).

Clicking a different column header will change the sorting to the respective column. .

Column resizing

Column width can be adjusted by dragging the separator lines between the columns. The separator line should appear on mouseover when the cursor is placed near the column border.

Column context menu

Some tables may need advanced column functionality, such as “Remove column”, “Replace column”, Add column” etc. This can be placed in a context menu (accessed by right-clicking on the column header).

Drill down/sub-view

An arrow to the right side of the row indicates that the row has a sub-view/detailed view. Clicking anywhere on the row (except controls) will navigate to the related sub-view.

Selecting rows

Rows are selected by checking the checkbox to the left side of the row. If the row has no sub-view, clicking anywhere on the row will check the checkbox and select the row. In matrix tables, checkboxes for selecting rows should be placed in a separate column.

Inline controls

Some tables may contain no controls, some contain a few inline controls, while others are matrices consisting solely of controls.

All inline controls are standard ABB Common UI controls, and fill the entire height of the row.

Row tracking

In some cases, it can be useful to highlight an entire row as the user is interacting with a table.

Highlight unsaved changes

In some cases it is useful to highlight modified fields to reduce the risk of user input errors. These examples highlight unsaved changes in blue.

Edit mode

In cases where the normal mode of interaction is monitoring and rarely editing, introducing an edit mode might be beneficial. A separate edit mode is a good way of preventing users from making unwanted changes to a table.

In this example the default table is read-only. Pressing “Edit” makes the cells editable. Pressing “Save” or “Discard changes” exits the edit mode.

Scroll and pagination

Consider the following when deciding on how to handle long lists of data:

  • The usability of long lists is primarily defined by the speed and performance of data fetching. Be sure to scale the data fetching architecture according to realistic usage patterns.
  • Browsing a list of items and comparing data is easier with scrolling than with pagination.
  • Pagination results in easily accessible page breaks, beginning and end points. It’s quicker to navigate directly to specific locations in the list, than with scrolling.
  • Be careful combining scrolling and pagination in the same list.
