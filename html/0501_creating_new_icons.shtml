Creating new icons

The ABB icon library is under constant development, and new icons will be continuously added. The idea is that the basic icon library should contain the most commonly used icons. In many cases, however, the icon needed for an application is very specific, and must be created from scratch. Keep these four steps in mind when creating new icons:

1Decide what the new icon should communicate

2Decide on visual style and harmonize with existing icons

3Ensure pixel sharpness and visual balance

4Test the icons on representative users to verify intuitiveness

1: Decide what the new icon should communicate

Which image or metaphor do you want to use?

  • Always strive to follow excisting conventions
  • Simplify - what is the core message?
  • Consider the context - are there already other icons in the interface you need to differentiate from or harmonize with?

Do not use obscure metaphors

Avoid obscure metaphors. Poor use of icons can be confusing rather than helpful, and degrade the user experience.

Do not try to communicate too much

Sometimes the message you want to convey is complex, and designing an unambiguous icon might be difficult. Avoid adding too much detail, ABB icons should be simple and communicate a clear message. Sometimes adding a label or simply replacing the icon with a text label might be a better solution.

2: Decide on visual style and harmonize with existing icons

The ABB icon philosophy is to use precise, simple and angular icons, that convey a clear and consistent message. Icons reflect the general ABB visual style. Make sure that new icons follow the same philosophy by:

Ensuring visual harmony with existing icons:

  • Can they be used as a starting point for new ones?
  • Use consistent line thickness and design elements for all icons.

Following these visual guidelines:

Straight and angular shapes

Use rounded corners only to represent physical shapes, not for styling. For instance, it is natural to use rounded corners on organic objects.

Solid fill

Solid icons are preferred, unless outlines better describe the shape (Ex. paper clip, magnifying glass). Some icons may exist both in solid and outline versions, for use in different contexts.

Perspective

Avoid use of perspective, except when neccessary to define the shape.

Orientation

Orientation sometimes helps to communicate a shape or an action. Limit the orientation of the icon to horizontal, vertical and 45 degree angles.

Secondary elements

Place secondary elements, such as plus or minus signs, over the lower right corner of the icon, inside the main icon grid.

Badges

Place badges over the top right corner of the icon, extending outside the main icon grid.

Color

Minimize color usage, and make sure to provide sufficient color contrast.

3: Ensure pixel sharpness and visual balance

The key to a good icon and a consistent icon library is using a precise grid and aim for well-balanced visual weight.

Main icon grid

All 3 icon sizes are based on a grid, to facilitate consistency and establish a clear set of rules for the positioning of graphic elements. The grid has a minimum 1 px transparent border. Each icon takes up a certain area of the grid, but for them to view in a uniform way vertically and horizontally next to each other, they may need to fill out the grid differently. A good example of this is a circle and a square - visually they look uniform, but the circle is in fact taller and wider.

Pixel grid

Always align to pixel grid. Position icons “on pixel” – meaning the X and Y coordinates are integers and do not contain decimals. Move graphics one px left or right if it is impossible to center align them “on pixel”.

Shapes and visual balance

Compare the shape and size of new icons with existing ones. Basic geometric shapes must typically fill out the grid differently to achieve well-balanced visual weight.

Some icons are by nature smaller than others, for example the ‘Play’ icon, ‘Close’ icon and arrows. If maximized to fit the size of the grid, these icons will appear much larger than the rest of the icons.

The icon density also influences the visual weight. Condensed, saturated icons may need to be smaller to maintain the visual balance.
